Tuplets

MusicXML uses two separate concepts: a <time-modification> element to encode the tuplet’s ratio and a <tuplet> element to encode the engraved tuplet itself. These go inside <note> elements. Parsers need to keep track of currently open tuplets and look for the value stop.

MNX uses a single <tuplet> element, which contains the notes or rests therein. The inner attribute encodes the rhythmic duration of the tuplet’s elements as if the tuplet were ignored; the outer attribute encodes the rhythmic duration taking the tuplet’s ratio into account.
