Octave shifts (8va)

In MusicXML, you put <octave-shift> elements around the notes that are affected by the octave shift; the issue of whether the octave shift affects the first note following the closing <octave-shift> is interpreted differently by different programs.

In MNX, you put a single <octave-shift> element just before the first note it affects. The element’s end attribute specifies the measure location of the last note for which the octave shift applies. In this example, the octave shift’s final note, the C quarter note, has measure location 2:1/2 (measure index 2, metrical position one half note into the bar).

In both MusicXML and MNX, pitches for notes within an octave shift are specified taking the octave shift into account (see MNX’s definition of “written pitch”). In this example, the final two pitches are C7 and C6, despite the notes being rendered on the same staff line.
