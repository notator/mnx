Part Groups

MusicXML supports two ways of organizing parts:

  • If the top-level element is <score-partwise>, then the markup must contain <part> elements that contain <measure> elements.
  • If the top-level element is <score-timewise>, then the markup must contain <measure> elements that contain <part> elements.

MNX first defines <part> elements, in a way similar to MusicXML’s <score-partwise>. (Each <part> contains <measure> elements.) It then defines <systemLayout> elements that are nested inside <score> elements. This example shows how an MNX file can contain information for printing both a full score and its parts from the same <part> information.

In MusicXML, metadata such as the title lives within a <part-list> element. In MNX, metadata goes in the individual <score> elements.

Note that all references to an internal id consist of a '#' character followed by the id string.
