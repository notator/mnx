MusicXML supports two ways of organizing parts:

  • If the top-level element is <score-partwise>, then the markup must contain <part> elements that contain <measure> elements.
  • If the top-level element is <score-timewise>, then the markup must contain <measure> elements that contain <part> elements.

MNX has only one way, similar to MusicXML’s <score-partwise>. Each <part> contains <measure> elements.

In MusicXML, part metadata such as the title lives within a <part-list> element. In MNX, part metadata goes in the individual <part> element.

Note that all references to an internal id consist of a '#' character followed by the id string.
